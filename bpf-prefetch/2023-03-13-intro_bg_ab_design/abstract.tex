Recent work shows that different applications have distinct memory access patterns, and no single prefetcher can improve all applications’ performance. While we can design a more smart prefetching algorithm for a specific group of applications, it may not meet other applications’ requirements. Current Linux kernel prefetching logic isn’t complex at all as it aims to benefit as many programs as possible, mainly focusing on sequential patterns. The kernel counts the cache hits between two page faults and uses this count to dynamically change the number of consecutive pages it wants to prefetch next. However, supporting multiple prefetchers in the Linux kernel isn’t easy. Fortunately, eBPF provides us a more secure and flexible framework to write different prefetchers as offloaded eBPF programs which can be dynamically hooked in Linux kernel at runtime.
